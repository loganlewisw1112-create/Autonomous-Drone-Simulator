Section 1Instructor path — live class on your LAN

Follow these steps on one Windows instructor PC. Students use other Windows laptops or desktops on the same Wi‑Fi — not phones.

  1. Install once — Node.js 20+, then:

    git clone https://github.com/loganlewisw1112-create/Autonomous-Drone-Simulator.git
cd Autonomous-Drone-Simulator
npm install

  2. Launch the desktop classroom app (preferred):

    npm run classroom:desktop

    Or PowerShell: pwsh scripts/windows/Start-ClassroomDesktop.ps1

  3. At the splash, choose Yes to start the Classroom Server. Electron starts the LAN relay and opens the classroom UI. Leave the app open.

  4. On Classroom home, create or sign in as an Instructor account.

  5. Open Start a training class. New instructors: type the school access code once → Finish account setup. (First code on a fresh machine becomes the school unlock automatically.)

  6. Choose scenario and seed → Create class. Note the 6-character class code.

  7. Show students the LAN address from the relay (or desktop status), for example http://192.168.x.x:8080 — not localhost on their machines.

  8. Students open that address on a Windows PC, create/sign in as Student, enter the class code, and join.

  9. Watch the instructor wall; end the class from the console when finished. Progress archives into the instructor’s encrypted on-device history.

  10. Quit the desktop app when done — that stops a relay this app started.

Terminal alternative

npm run classroom builds and starts the relay in a terminal instead of Electron. Leave that window open for the whole class. Prefer the desktop app when you want Yes/No and quit-to-stop.

Section 2What you need

RoleMachineNotes
Instructor Windows laptop or desktop Node 20+, this repo, same Wi‑Fi as students. Stays on for the session.
Students Windows laptop or desktop Modern browser only — no phone/tablet join. Same Wi‑Fi (not guest isolation).

Practical expectations: instructor PC with a working Wi‑Fi/Ethernet adapter and enough RAM for Electron + the relay (typical classroom laptops are fine). First npm install downloads Electron and needs network. No signed installer yet — run from the repo with the npm scripts above.

Windows only

Classroom entry is gated to Windows. Phones and tablets see WINDOWS CLASSROOM ONLY and cannot join a live class. Solo practice outside class can use the Mobile sim build.

Section 3Instructor unlock — type a code only

Creating a new instructor account needs a one-time access code on Start a training class.

  • You only type a code in the UI. Never create folders or paste hex digests by hand.
  • On a fresh machine, the first code typed becomes the school unlock (saved in browser storage and, when the LAN relay is up, under gitignored local-secrets/ automatically).
  • If a school code is already set, later instructors must enter that same code — the app will not silently overwrite it.
  • Optional plaintext recovery may appear as local-secrets/instructor-access-code.txt on the instructor PC. You do not need to create or edit that file.
Reset (admin only)

Clear this site’s browser storage for the classroom origin, and either delete local-secrets/instructor-access-hash.txt (and the optional code file) or call DELETE /api/instructor-access on the relay. Then type a new first code.

Hand the unlock code to instructors offline. Do not put it in README, git, or public chat. Students never need the unlock code — only instructors do.

Section 4Desktop app — Yes / No / Quit

Preferred launch: npm run classroom:desktop (or the PowerShell helper). After build, Electron owns the relay.

ChoiceWhat happens
Yes Starts server/classroom.mjs if it is not already up, opens the classroom UI, keeps the relay alive until you quit.
No Connects if a relay is already running; otherwise shows short setup and lets you open the UI without a live server.
Quit app Stops a relay this app started (Windows process tree). Does not kill a relay you started separately in a terminal. 
npm run classroom:desktop          # build + open Electron
npm run classroom:desktop:launch   # Electron only (dist/ already built)
npm run classroom                  # terminal: build + relay (leave open)
npm run classroom:relay            # restart relay without rebuild

Section 5Web / Vercel / GitHub demo only

The hosted classroom link is a client showcase. Browsers and Vercel cannot spawn Node, so they cannot start the LAN relay.

  • On web, the Yes button only probes for an already-running server — it does not start one.
  • No = short setup copy, then continue into UI without a live multi-student class.
  • Hosted unlock may still use a dashboard env digest when present; operators still only type a code in the UI.
Honest limit

A real multi-student class needs one Windows instructor PC running npm run classroom:desktop (or npm run classroom) on the same Wi‑Fi as the students. Vercel alone is not enough.

Section 6Missions & operator notes

Catalog: 25 incident missions (10 historical + 15 training/refreshed) plus 6 NIST skills drills outside those 25. Same seed → same mission for fair comparison.

Brief operator-relevant notes

  • Auth training — Preflight AUTH steps (RID / LAANC-style / ceiling / TFR as authored). Launch stays gated until required steps are done. Simulation only — no real FAA network.
  • Exact thermal contacts — Reported contact coordinates match heat-source truth (detection still gated by range, NETD, atmosphere, LOS).
  • Replay — Recording stops at ~25 minutes; it does not drop oldest frames in a rolling window.
  • Mobile command base — Moving the CB before launch replans parked drone bays around the new position (all builds).
  • Terrain — Priority AOs ship DEM fixtures; desktop/Windows/classroom can show MapLibre terrain. Mobile keeps a flat map.
  • Radiometric thermal — Payload specs (Boson+ / Hadron 640R class) drive Johnson Rd and sensor HUD; ±5 °C accuracy is metadata only, not a full IR image sim.

Live class traffic is end-to-end encrypted to the instructor’s browser key. The relay routes ciphertext only. Refreshing the instructor tab ends that class — leave it open.

Section 7During / after class

The instructor wall shows one live tile per student (map dots, time, battery, alerts). Click a tile to focus one student for a denser stream. Critical tiles sort to the front.

When a run ends, encrypted results appear in the results pane (including Auth% / AuthMissed when authorization steps apply). Export CSV downloads the table. Ending the class archives per-student progress into the instructor’s on-device encrypted classroom history.

Join address

Write the LAN base URL on the board before class. Students can open the home page and use Join, or go straight to ?join=<CODE>. Do not tell them to open localhost on another machine.

Section 8Troubleshooting

SymptomCause and fix
Server not up / Yes finds nothing (web) Web Yes only probes. Start the desktop app with Yes, or run npm run classroom on the instructor PC, then retry.
Students cannot reach the address Firewall (allow Node/Electron on private networks), same Wi‑Fi (not guest isolation), students using the printed LAN IP — not localhost.
Wrong OS / phone blocked Classroom is Windows-only. Use a Windows laptop/desktop, or solo Mobile for practice outside class.
Access code rejected / want a new school code Enter the existing school code, or do an intentional admin reset (clear site storage + delete local-secrets/instructor-access-*.txt or DELETE /api/instructor-access), then type a new first code.
"No class with that code is running." Wrong code or class ended. Codes have no vowels — a reported "O" is zero.
Instructor refreshed or closed the console tab That class’s key lived only in the tab. Create a new class and read out the new code. Prefer quitting via the desktop app when ending for the day.
Ordinary simulator instead of classroom UI Rebuild with classroom mode: npm run classroom:desktop or npm run classroom (not a plain npm run build).
Class full / relay max classes Cap is 40 students per class, 8 classes per relay. End a class or start another on the same relay.

Section 10Safety

Simulation only

This product is a safety-oriented search-and-rescue / public-safety training simulator. It does not connect to real drones, cameras, dispatch systems, Remote ID hardware, or FAA/LAANC networks. Label demos as simulation. Keep use cases inspection, logistics, SAR, operations, and review — never weapon targeting or harmful deployment.
